Implementing an algorithm for finding the least resource intensive path using reinforcement learning (Q-Learning)

When the reinforcement learning algorithm is launched, an array of possible states and actions R is created, which shows where the agent can go and where not, as well as the location of the target on the map (the numbering starts at zero and runs horizontally and top down in ascending order). After that, the second array Q is created and filled with zeros. Then training takes place: in order for an agent to learn how to find the lease resource intensive path from any position on the map, it needs to check each such state (after creating the first array R, an array of possible initial states is also created).

Then the agent leaves each such state and calculates his next move, comparing the rewards for all possible subsequent actions. After that, the second array with award weights is filled in accordance with formula (3) (Markov decision process):

Q(s,a) = r(s,a) + gamma maxQ(s',a)

where s - the current state, a - the next action, gamma - the discount coefficient. It is used to balance the immediate and future rewards. Typically, this value ranges from 0.8 to 0.99. The discount coefficient controls the speed of learning, the more this parameter is, the faster the neural network learns, but the more errors it can make. s’ is the next state after the next action. The maxQ (s’, a) function returns the maximum reward for an available action from state s’. After the training is completed, the second array Q stores the rewards of the possible actions for each state.

The User can enter any initial state and get the least resource intensive path. Such an algorithm can be used for devices that operate on static map, for example, a robot vacuum cleaner. At the first scan of the apartment, the algorithm will compose the R array and will be trained, and on subsequent trips, the vacuum cleaner will already have a ready array of Q rewards in it that will allow it to instantly get the least resource intensive path to the goal (dock-station) from any starting location, which will also improve with every departure.

Based on the results of this stage, it was decided, when solving the problem of finding the shortest path, to use both the described learning algorithms, since in the aggregate they cover almost all possible search needs and environmental conditions.
